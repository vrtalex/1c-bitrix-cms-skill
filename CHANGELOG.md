# Changelog

Все заметные изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [1.0.5] - 2026-06-30

### Добавлено

- **Каталог штатных компонентов** (`shared/kb/components/`) — индекс + карточки по
  доменам (новости, каталог, корзина, заказ, навигация, формы/авторизация, поиск):
  назначение, ключевые `arParams`, типовой вызов, что кастомизировать в `/local`,
  типовые ошибки. Помогает собирать страницы из штатных компонентов и переопределять
  их шаблоны в `/local`.
- **Модель «право исполнять» (E1/E2/E3)** в конвенциях: чтение/диагностика — везде;
  обратимая запись — в `/local`; необратимые/привилегированные операции (обновления,
  восстановление, откат) — не на боевом сайте по умолчанию (инструкция человеку или
  изолированная копия). На соответствующих рецептах — пометка класса исполнения.
- **Локальный контейнерный dev-стенд** (env-docker) как среда для разработки,
  тестирования и проверки рискованных операций (в отличие от боевого сайта).
- **Паттерн верификации результата** (operations): структура данных — через
  CLI-интроспекцию, рендер страницы — через браузер; на dev-стенде.

### Изменено

- База знаний помечена как самодостаточная: ссылки указывают происхождение факта и
  по умолчанию не требуют вычитывания первоисточников; убраны малоинформативные
  ссылки на корни сайтов вендора (провенанс и точные ссылки сохранены).
- Добавлен список нежелательного API в версионный baseline; внутренняя статическая
  проверка примеров в рецептах.

## [1.0.1] - 2026-06-28

### Добавлено

- **Установщик одной командой** (`install.sh` / `install.ps1`): авто-определение
  Claude Code и Codex, установка пакета с сохранением раскладки, проверка версии
  и обновление.
- **Автоматизация релизов**: проверки пакета и публикация релизов из `VERSION` +
  `CHANGELOG`.
- **Машиночитаемый baseline платформы** (`version-baseline.json`): минимальные
  версии PHP/СУБД и снятые на новых ядрах API.
- **Расширение под-скилла обновлений** (5 → 9 рецептов): диагностика проблем
  обновления (каталог симптом → причина → фикс), миграция кодировки cp1251 →
  UTF-8, обновление решений Marketplace / партнёрских / собственных модулей,
  безопасное тест-обновление на копии.
- **Расширение SEO** (5 → 10 рецептов): дубли и коды ответа, SEO умного фильтра
  каталога, скорость и Core Web Vitals, пагинация, hreflang / мультиязычность.

### Изменено

- **Усиление безопасности**: проверка подписи платёжных callback по шлюзам,
  рекомендации 152-ФЗ для форм, контроль целостности после инцидента, выявление
  привилегированных установщиков в публичной зоне; статический `check-conventions`
  дополнен и получил строгий режим для CI.
- **Актуализированы версии и совместимость**: PHP 8.2+, MySQL 8.0+ / utf8mb4.

### Состав

- **75 рецептов**, 12 скиллов.

[1.0.1]: https://github.com/vrtalex/1c-bitrix-cms-skill/releases/tag/v1.0.1

## [1.0.0] - 2026-06-28 — Первый публичный релиз

### Добавлено

- **12 скиллов**: оркестратор `1c-bitrix-cms` и 11 специализированных под-скиллов
  (content, template, seo, commerce, settings, setup, deploy, update, rest,
  security, quality), покрывающих полный цикл работы с сайтом на
  «1С-Битрикс: Управление сайтом» (ядро 26.x).
- **66 рецептов** по всему циклу разработки: управление контентом, информационные
  блоки, шаблоны сайта и компонентов, SEO, интернет-магазин, деплой, обновления
  платформы, интеграции через REST, безопасность и контроль качества кода.
- **Встроенные правила безопасности** и threat-intel: проверенные практики
  защиты от типовых уязвимостей и актуальные индикаторы компрометации.
- **Статический `check-conventions`**: проверка кода на соответствие
  соглашениям платформы без запуска проекта.
- **Поддержка Claude и Codex**: скиллы работают в обоих агентских окружениях.
