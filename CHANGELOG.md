# Changelog

Все заметные изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [1.0.6] - 2026-06-30

### Добавлено

- **Модель установки «с нуля» в под-скилле setup.** Рецепт-оркестрация
  ([03-installation.md](../shared/kb/recipes/setup/03-installation.md)) выбирает
  один из трёх режимов — готовая среда → сразу сборка контента; мастер установки;
  restore из чистого base-бэкапа — и завершает любой из них единым **гейтом
  завершённости** (вход в админку, открытие визуального редактора, публичный слой
  на месте, установочные скрипты удалены). До зелёного гейта сайт считается не
  установленным и контент не строится.
- **Рецепт мастера установки**
  ([06-install-wizard.md](../shared/kb/recipes/setup/06-install-wizard.md)) —
  карта полей мастера (welcome → agreement → check_license_key → requirements →
  create_database → create_modules → финал) и паттерн браузер-хэндоффа: агент
  проходит детерминированные шаги сам, человека привлекает только там, где нужно
  (реальный ключ, согласие с лицензией, пароль администратора).
- **Воспроизводимое развёртывание из чистого base-бэкапа**
  ([deploy/03-backup-restore.md](../shared/kb/recipes/deploy/03-backup-restore.md))
  — `backup:create` чистой базы → restore на новом окружении → удаление
  установочных скриптов → гейт. Уточнено, что CLI (`bitrix.php`,
  [04-cli-console.md](../shared/kb/recipes/setup/04-cli-console.md)) — для
  разработки и обслуживания уже установленного сайта, не для первичной установки.
- **`detect-bitrix.sh`: сигнал `install_finalized`** (`Y`/`N`/`unknown`) —
  различает доведённую до конца установку и недо-финализированную базу, чтобы
  оркестратор выбрал режим без угадывания.
- **`check-conventions.sh`: правило на оставленные установочные/служебные скрипты**
  (`bitrixsetup.php`, `restore.php`, `bx_1c_import.php`, `bitrix_server_test.php`,
  `bitrix/install/`) — Сканер безопасности вендора флагует их как угрозу; чекер
  напоминает удалить их перед сдачей. Добавлен регрессионный тест.

### Изменено

- **env-docker** обозначен как рекомендованная вендором среда для небоевого
  использования (разработка, тестирование, код-ревью, проверка безопасности).
- **Паттерн верификации результата** (структура — CLI-интроспекцией, рендер —
  через браузер, на dev-стенде) распространён на секции «Проверка» публично-
  выводящих рецептов (контент, шаблоны, SEO, витрина магазина) — единой ссылкой
  на общий раздел `operations`.

### Исправлено

- **`check-conventions.sh`: правило обнаружения SQL-инъекций** снова работает.
  Шаблон, начинающийся с `->` (`->Query`/`->Execute`), теперь передаётся в `grep`
  через `-e` — иначе `grep`/`ugrep` принимал его за опцию, правило молча
  отключалось. Добавлен регрессионный тест (выявлено прогоном на живом ядре).

- **Аудит модели установки — исправления.** По итогам сквозного разбора
  механик установки:
  - **`detect-bitrix.sh`: `install_finalized`** ловит оставленный `restore.php`
    по суффиксу (включая префиксный `<домен>.restore.php` и регистр) и
    карантинный суффикс `*.suspected` — раньше пропускал и отдавал ложный `Y`;
    маркер инсталлятора (`BX_BITRIX_INSTALL`) распознаётся в обоих вариантах
    кавычек; эвристика кодировки читает многострочный `utf_mode`.
  - **`check-conventions.sh`:** добавлено правило на веб-шелл
    (`eval()/assert()/динамический вызов` на данных запроса); правила с обходом
    дерева больше не ломаются на путях с пробелами (NUL-устойчивый перебор);
    правило оставленных установщиков расширено (`*restore.php`, `*.suspected`);
    правило секретов не глушится плейсхолдером из комментария.
  - **Гейт завершённости** ключается на файл `bitrix/install/index.php`, а не на
    весь каталог `bitrix/install/` (он остаётся — рантайм-ассеты ядра), —
    согласовано со скриптами; убран ложно-красный гейт.
  - **Класс исполнения по режимам:** Режим 3 (restore) и установка поверх
    существующего окружения помечены E3 (необратимая перезапись), Режим 1 и
    Режим 2 на пустом окружении — E2.
  - **Развёртывание/восстановление:** заменён несуществующий
    `bitrix.php backup:create` на штатный `backup.php`/мастер админки; добавлены
    ограничение доступа к `restore.php` на время окна, HTTPS и сверка
    контрольной суммы, закрытие архивов от HTTP и их удаление в очистке,
    безусловный fallback-удаления.
  - **dev-среда:** обязательное обезличивание прод-дампа (хеши паролей, ПДн,
    живые секреты) до использования копии; `ONLY_EMAIL` обозначен как перехват
    почты, а не санация; реальный контроль доступа к стенду (не `robots.txt`);
    `BX_UTF` убран из отладочного блока; запрет `debug` на проде.

## [1.0.5] - 2026-06-30

### Добавлено

- **Каталог штатных компонентов** (`shared/kb/components/`) — индекс + карточки по
  доменам (новости, каталог, корзина, заказ, навигация, формы/авторизация, поиск):
  назначение, ключевые `arParams`, типовой вызов, что кастомизировать в `/local`,
  типовые ошибки. Помогает собирать страницы из штатных компонентов и переопределять
  их шаблоны в `/local`.
- **Модель «право исполнять» (E1/E2/E3)** в конвенциях: чтение/диагностика — везде;
  обратимая запись — в `/local`; необратимые/привилегированные операции (обновления,
  восстановление, откат) — не на боевом сайте по умолчанию (инструкция человеку или
  изолированная копия). На соответствующих рецептах — пометка класса исполнения.
- **Локальный контейнерный dev-стенд** (env-docker) как среда для разработки,
  тестирования и проверки рискованных операций (в отличие от боевого сайта).
- **Паттерн верификации результата** (operations): структура данных — через
  CLI-интроспекцию, рендер страницы — через браузер; на dev-стенде.

### Изменено

- База знаний помечена как самодостаточная: ссылки указывают происхождение факта и
  по умолчанию не требуют вычитывания первоисточников; убраны малоинформативные
  ссылки на корни сайтов вендора (провенанс и точные ссылки сохранены).
- Добавлен список нежелательного API в версионный baseline; внутренняя статическая
  проверка примеров в рецептах.

## [1.0.1] - 2026-06-28

### Добавлено

- **Установщик одной командой** (`install.sh` / `install.ps1`): авто-определение
  Claude Code и Codex, установка пакета с сохранением раскладки, проверка версии
  и обновление.
- **Автоматизация релизов**: проверки пакета и публикация релизов из `VERSION` +
  `CHANGELOG`.
- **Машиночитаемый baseline платформы** (`version-baseline.json`): минимальные
  версии PHP/СУБД и снятые на новых ядрах API.
- **Расширение под-скилла обновлений** (5 → 9 рецептов): диагностика проблем
  обновления (каталог симптом → причина → фикс), миграция кодировки cp1251 →
  UTF-8, обновление решений Marketplace / партнёрских / собственных модулей,
  безопасное тест-обновление на копии.
- **Расширение SEO** (5 → 10 рецептов): дубли и коды ответа, SEO умного фильтра
  каталога, скорость и Core Web Vitals, пагинация, hreflang / мультиязычность.

### Изменено

- **Усиление безопасности**: проверка подписи платёжных callback по шлюзам,
  рекомендации 152-ФЗ для форм, контроль целостности после инцидента, выявление
  привилегированных установщиков в публичной зоне; статический `check-conventions`
  дополнен и получил строгий режим для CI.
- **Актуализированы версии и совместимость**: PHP 8.2+, MySQL 8.0+ / utf8mb4.

### Состав

- **75 рецептов**, 12 скиллов.
