# Установка — 1c-bitrix-cms

Навык подключается одинаково для **Claude Code** и **Codex**. Самый быстрый путь — установщик одной командой: он сам разложит 12 каталогов навыков и базу знаний в нужную раскладку.

## Установка одной командой

### macOS / Linux (bash)

```sh
curl -fsSL https://raw.githubusercontent.com/vrtalex/1c-bitrix-cms-skill/main/install.sh | bash
```

По умолчанию (режим `--auto`) установщик находит все доступные homes — `~/.claude` и/или Codex (`$CODEX_HOME`, иначе `~/.codex`) — и ставит навык в каждый.

### Windows (PowerShell)

```powershell
irm https://raw.githubusercontent.com/vrtalex/1c-bitrix-cms-skill/main/install.ps1 | iex
```

## Флаги установщика

Одинаковы для `install.sh` (в виде `--flag`) и `install.ps1` (в виде `-Flag`):

| Флаг | Назначение |
|---|---|
| `--auto` | во все обнаруженные homes (по умолчанию) |
| `--claude` | только в `~/.claude` |
| `--codex` | только в `$CODEX_HOME` (иначе `~/.codex`) |
| `--both` | и в Claude, и в Codex |
| `--version X.Y.Z` | установить конкретный релиз (по умолчанию — последний) |
| `--local` | копировать из текущего клона (`./skills` и `./shared`), без обращения к сети |
| `--check` | вывести `UPDATE_AVAILABLE local=X remote=Y` или `UP_TO_DATE` и выйти (ничего не меняет) |
| `--dry-run` | показать планируемые операции без изменений на диске |
| `-h`, `--help` | справка |

Примеры:

```sh
# bash
bash install.sh --both
bash install.sh --claude --version 1.0.0
bash install.sh --check                 # есть ли обновление
```

```powershell
# PowerShell
.\install.ps1 -Both
.\install.ps1 -Claude -Version 1.0.0
.\install.ps1 -Check
```

Установка из локального клона (например, для офлайна или своей сборки):

```sh
git clone https://github.com/vrtalex/1c-bitrix-cms-skill
cd 1c-bitrix-cms
bash install.sh --local --both
```

## Раскладка после установки

Установщик кладёт каждый из 12 навыков в `<HOME>/skills/<имя-навыка>/`, а базу знаний — в соседний `<HOME>/shared/`. Это важно: под-скиллы адресуют знания по относительным путям `../../shared`, и из `<HOME>/skills/<навык>/SKILL.md` путь `../../shared` указывает ровно на `<HOME>/shared`.

```
<HOME>/                                 (~/.claude или ~/.codex)
├── skills/
│   ├── 1c-bitrix-cms/SKILL.md          ← оркестратор (точка входа)
│   ├── 1c-bitrix-cms-content/SKILL.md
│   ├── 1c-bitrix-cms-commerce/SKILL.md
│   └── …                                ← остальные под-скиллы (всего 12)
└── shared/
    ├── kb/                              ← база знаний + 66 рецептов
    ├── assets/                          ← скелеты компонентов/страниц/шаблонов
    └── scripts/check-conventions.sh     ← статическая проверка
```

`skills/` и `shared/` обязательно соседи: под-скилл обращается к знаниям как `../../shared/kb/...`. Установщик автоматически проверяет это после копирования — что орхестратор на месте и что `../../shared/kb/00-overview.md` разрешается из каталога навыка.

## Проверка, что навык загрузился

- Спросите агента: «Какие под-скиллы 1c-bitrix-cms доступны?» — он должен перечислить оркестратор и 11 доменных под-скиллов (setup, settings, template, content, seo, commerce, deploy, security, quality, update, rest).
- Дайте задачу из примеров README, например: «Выведи каталог через `bitrix:catalog.section` с кэшем.» Агент должен сослаться на рецепт из `shared/kb/recipes/` и держать инварианты (код в `/local`, ядро не трогать).
- Проверьте, что статическая проверка доступна:

```sh
sh shared/scripts/check-conventions.sh <каталог_проекта>
```

Скрипт выводит `[FAIL]` для надёжных нарушений и `[REVIEW]` для эвристик безопасности, требующих ручного разбора.

## Обновление

Узнать, есть ли новая версия (без изменений на диске):

```sh
bash install.sh --check
```

Если вывод `UPDATE_AVAILABLE`, обновитесь той же командой установки — установщик ставит файлы атомарно (через временный каталог) и заменяет прежнюю версию.

## Установка вручную (запасной вариант)

Если установщик неприменим, разложите файлы руками — сохранив раскладку «соседи `skills/` и `shared/`».

### Claude Code

1. Скопируйте каталоги `skills/` и `shared/` так, чтобы они остались соседями: глобально — `~/.claude/skills/` и `~/.claude/shared/`; либо в рамках проекта — `<project>/.claude/skills/` и `<project>/.claude/shared/`.
2. Запустите Claude Code в каталоге вашего Битрикс-проекта.
3. Навык подхватывается в начале сессии. Оркестратор `1c-bitrix-cms` срабатывает, когда вы упоминаете работу с сайтом на 1С-Битрикс (инфоблоки, компоненты, каталог, шаблон и т. п.).

### Codex

1. Разместите `skills/` и `shared/` (соседями) в каталоге навыков вашей установки Codex (`$CODEX_HOME/skills` и `$CODEX_HOME/shared`, по умолчанию `~/.codex`).
2. Запустите Codex в каталоге проекта.
3. Навык активируется в начале сессии; маршрутизация к под-скиллам — через оркестратор `1c-bitrix-cms`.

## Требования к окружению

- ядро «1С-Битрикс: Управление сайтом» **26.x** (выверено на 26.150.0);
- **PHP ≥ 8.2** (рекомендуется 8.4+);
- **MySQL ≥ 8.0** / **utf8mb4** (на Linux альтернатива — MariaDB 10.x/11.x); только UTF-8;
- веб-сервер Apache 2.0+ или Nginx;
- для установщика `install.sh` — `curl` и `tar`; для `check-conventions.sh` — POSIX-совместимый shell.

Актуальные системные требования вендора: [dev.1c-bitrix.ru/user_help/reqintro.php](https://dev.1c-bitrix.ru/user_help/reqintro.php).

Навык работает в любой из сред: «живой Битрикс» (есть `bitrix/` и доступ к БД — можно выполнять код и проверять результат), «только файлы» (БД недоступна — агент создаёт корректные файлы и даёт инструкцию запуска), удалённый доступ (FTP/SSH/REST) и чистый старт (Битрикса ещё нет — начинается с под-скилла `1c-bitrix-cms-setup`).
