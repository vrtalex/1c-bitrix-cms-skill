# 1c-bitrix-cms

**AI-навык, превращающий Claude и Codex в опытного разработчика на «1С-Битрикс: Управление сайтом».**
*Make Claude & Codex write correct, secure, convention-clean 1C-Bitrix code.*

`12 skills` · `80 recipes` · `Claude + Codex` · `source-available`

---

## Что это

Это упакованная база знаний и набор пошаговых рецептов по платформе «1С-Битрикс: Управление сайтом» (ядро 26.x), оформленные как AI-навык (skill). Когда навык подключён, ваш агент — Claude Code или Codex — следует принятым на платформе практикам: пишет код в `/local`, не трогает ядро, применяет правила безопасности и выбирает подходящий API под задачу. Знания опираются на изучение ядра и официальную документацию вендора.

## Зачем

Без подсказки агент угадывает имена API, сигнатуры компонентов и конвенции проекта — и легко промахивается мимо того, как платформа устроена на самом деле. Этот пакет даёт ему верные ориентиры:

- весь свой код — в `/local`, **ядро не модифицируется** (обновления и совместимость остаются в силе);
- **безопасность по умолчанию** — защита от CSRF/XSS/SQLi, аккуратная работа с секретами и загрузкой файлов;
- **верный выбор D7 vs legacy** под конкретную задачу — каждый рецепт указывает рекомендованный путь;
- предсказуемая структура файлов компонентов, шаблонов и модулей.

Результат — код, который сразу ложится в проект и проходит ревью, а не требует переписывания.

## Что внутри

Оркестратор `1c-bitrix-cms` определяет среду (живой Битрикс / только файлы / удалённый доступ / чистый старт), держит инварианты (код в `/local`, ядро не трогать, проверки перед сдачей) и маршрутизирует задачу к нужному под-скиллу.

| Под-скилл | Домен |
|---|---|
| `…-setup` | окружение, установка, структура проекта, CLI |
| `…-settings` | `.settings.php`, конфиг, события, агенты, кэш |
| `…-template` | шаблон сайта, вёрстка, меню, включаемые области |
| `…-content` | инфоблоки, компоненты 2.0, вывод контента, интроспекция проекта |
| `…-seo` | sitemap, ЧПУ, редиректы, мета-теги, robots, Open Graph |
| `…-commerce` | каталог, цены, заказы, оплата, доставка, обмен с 1С |
| `…-deploy` | сборка, выкладка, миграции, окружения |
| `…-security` | проактивная защита, безопасное кодирование, права, аудит, threat-intel |
| `…-quality` | конвенции, статические проверки, ревью кода |
| `…-update` | обновление ядра и модулей, контроль совместимости |
| `…-rest` | REST/вебхуки, интеграции, прототипирование |

**11 под-скиллов + оркестратор = 12 skills**, и **80 рецептов** по доменам — от создания инфоблока и вывода каталога до checkout, обмена с 1С и пост-инцидентного разбора.

В комплекте также:

- **статический `check-conventions`** — проверяет проект на нарушения конвенций и типовые промахи безопасности перед сдачей;
- **покрытие безопасности и threat-intelligence** — встроенные правила (CSRF/XSS/SQLi/секреты/загрузка файлов), известные CVE с точками входа, индикаторы компрометации и сигнатуры для контроля целостности.

## Быстрый старт

Установщик одной командой разложит 12 навыков и базу знаний в нужную раскладку и поставит их во все обнаруженные homes (`~/.claude` и/или Codex).

**macOS / Linux:**

```sh
curl -fsSL https://raw.githubusercontent.com/vrtalex/1c-bitrix-cms-skill/main/install.sh | bash
```

**Windows (PowerShell):**

```powershell
irm https://raw.githubusercontent.com/vrtalex/1c-bitrix-cms-skill/main/install.ps1 | iex
```

Можно выбрать цель и источник: `--claude` / `--codex` / `--both`, `--version X.Y.Z`, `--local` (из текущего клона), `--check` (есть ли обновление), `--dry-run`. После установки запустите Claude Code или Codex в проекте — навык подхватится автоматически; точка входа — оркестратор `1c-bitrix-cms`.

Флаги, проверка загрузки и установка вручную: [INSTALL.md](INSTALL.md).

## Для чего создан

Навык создан целенаправленно под **корпоративные сайты и интернет-магазины на «1С-Битрикс: Управление сайтом»** (редакция Бизнес, ядро 26.x). Фокус узкий и осознанный: глубокое знание именно этой платформы — её инфоблоков, компонентов 2.0, шаблонов, модулей `catalog`/`sale`, REST и подсистемы безопасности — вместо поверхностного покрытия «всего сразу». Для задач за пределами этого фокуса навык честно направляет к штатным мастерам и модулям админки.

## Безопасность и качество

- весь код — в `/local`, **ядро не модифицируется**;
- встроенные правила безопасности: CSRF (проверка `sessid`), XSS (`htmlspecialcharsbx`, `CBXSanitizer`, валидация схемы URL), SQLi (ORM/параметризация), безопасная загрузка файлов, аккуратное обращение с секретами;
- **threat-intel и контроль целостности** — известные уязвимости с точками входа, IOC и сигнатуры для проверки логов и файлов;
- **статический `check-conventions`** прогоняется перед сдачей; надёжные нарушения помечаются как `[FAIL]`, эвристики безопасности — как `[REVIEW]` для ручного разбора.

**Честно об охвате.** База знаний построена на глубоком изучении платформы и официальной документации вендора. Ключевые исполнимые домены проверены на живом Битрикс 26.150.0; остальное опирается на статическую проверку и изучение ядра/документации. Перепроверяйте результаты под своё окружение и версию ядра.

## Требования

- ядро «1С-Битрикс: Управление сайтом» **26.x** (выверено на 26.150.0);
- **PHP ≥ 8.2** (рекомендуется 8.4+);
- **MySQL ≥ 8.0** с кодировкой **utf8mb4** (альтернатива на Linux — MariaDB 10.x/11.x); только UTF-8;
- веб-сервер Apache 2.0+ или Nginx.

Актуальные системные требования вендора: [dev.1c-bitrix.ru/user_help/reqintro.php](https://dev.1c-bitrix.ru/user_help/reqintro.php).
