# Компоненты: поиск и прочее

Поиск по сайту — полные карточки; карты и подписка — тонкие заглушки-указатели
(настраиваются мастером/в админке, без сниппета). Поиск опирается на модуль
`search` и индексацию; рецепт — `../recipes/15-site-search.md`.

---

## `bitrix:search.title`

1. **Назначение.** Поле быстрого поиска по заголовкам с динамическими подсказками
   (выпадающий список совпадений по мере ввода).
2. **Когда брать.** Строка поиска в шапке сайта. Страницу с полной выдачей рисует
   `bitrix:search.page` — обычно `search.title` ведёт на неё по Enter.
3. **Ключевые `arParams`.**
   - `PAGE` — URL страницы результатов (`/search/`).
   - `NUM_CATEGORIES`, `TOP_COUNT` — сколько категорий/совпадений в подсказке.
   - `CATEGORY_0`, `CATEGORY_0_TITLE` — по каким областям искать (`iblock`, и др.).
   - `SHOW_INPUT` — показывать поле ввода.
   - `CHECK_DATES` — учитывать актуальность.
4. **Типовой вызов.**
   ```php
   $APPLICATION->IncludeComponent("bitrix:search.title", ".default", [
       "PAGE"          => "/search/",
       "NUM_CATEGORIES"=> 1,
       "TOP_COUNT"     => 5,
       "CATEGORY_0"    => ["iblock_news", "iblock_catalog"],
       "SHOW_INPUT"    => "Y",
   ]);
   ```
5. **Что кастомизируют в `/local`.** Шаблон — в
   `/local/templates/<tpl>/components/bitrix/search.title/<tpl>/template.php`.
   Правят разметку поля и выпадающего списка; JS подсказок штатный — контейнеры
   результатов сохранять.
6. **Типовые ошибки.**
   - Пустые подсказки — не запущена индексация поиска (модуль `search`).
   - `PAGE` ведёт не на страницу с `search.page` → Enter уходит в никуда.
   - Категории `CATEGORY_0` не охватывают нужные инфоблоки.
7. **Источник.** [src](https://dev.1c-bitrix.ru/user_help/components/sluzhebnie/search/search_title.php)

## `bitrix:search.page`

1. **Назначение.** Страница результатов поиска по сайту: форма запроса, выдача с
   пагинацией, подсветка совпадений.
2. **Когда брать.** Отдельная страница `/search/`. В паре с `search.title` (поле в
   шапке отправляет запрос сюда).
3. **Ключевые `arParams`.**
   - `RESTART` — перезапуск поиска при пустом запросе.
   - `PAGE_RESULT_COUNT` — результатов на странице.
   - `CHECK_DATES` — учитывать актуальность.
   - `arrWHERE` / `SHOW_WHERE` — ограничение области поиска.
   - `DISPLAY_TOP_PAGER`, `DISPLAY_BOTTOM_PAGER` — пагинация.
   - `CACHE_TYPE`, `CACHE_TIME`.
4. **Типовой вызов.**
   ```php
   $APPLICATION->IncludeComponent("bitrix:search.page", ".default", [
       "RESTART"           => "N",
       "PAGE_RESULT_COUNT" => 20,
       "CHECK_DATES"       => "N",
       "DISPLAY_BOTTOM_PAGER" => "Y",
       "CACHE_TYPE"        => "A", "CACHE_TIME" => 3600,
   ]);
   ```
5. **Что кастомизируют в `/local`.** Шаблон — в
   `…/search.page/<tpl>/template.php`. Правят разметку выдачи и подсветку;
   `SetTitle`/крошку страницы поиска ставят на самой странице или в эпилоге.
6. **Типовые ошибки.**
   - Пустая выдача — индексация поиска не настроена/не запущена.
   - Поиск находит «всё подряд» — не ограничена область (`arrWHERE`).
   - Морфология/стемминг зависят от настроек модуля `search`, не от компонента.
7. **Источник.** [src](https://dev.1c-bitrix.ru/user_help/components/sluzhebnie/search/search_page.php)

## `bitrix:map.*` (заглушка — карты)

1. **Назначение.** Вывод интерактивной карты (Яндекс/Google) с метками — обычно на
   странице контактов.
2. **Когда брать.** Нужна карта на публичной странице. Компонент карты ставят и
   настраивают визуальным редактором/мастером (координаты, метки, API-ключ
   провайдера задаются в настройках компонента, а не кодом). Готового сниппета не
   приводим — это конфигурация в админке/редакторе.
7. **Источник.** [src](https://dev.1c-bitrix.ru/user_help/components/services/index.php) — раздел «Сервисы» документации компонентов.

---

## `bitrix:subscribe.*` (заглушка — подписка)

1. **Назначение.** Форма подписки на e-mail-рассылки (модуль `subscribe`):
   подписка/отписка, управление рубриками.
2. **Когда брать.** Блок «подписаться на новости». Рубрики рассылок и шаблоны
   писем настраиваются в админке (модуль `subscribe`); компонент формы ставят
   мастером. Сниппет не приводим — основная работа в админке рассылок.
7. **Источник.** [src](https://dev.1c-bitrix.ru/user_help/components/services/index.php) — раздел «Сервисы» документации компонентов.
