# Привязка нескольких шаблонов к сайту, темы и адаптивность

## Цель

К одному сайту привязать несколько шаблонов дизайна и управлять тем, какой из них применится на конкретной странице: настроить отсортированный список правил привязки по правилу «перевёрнутой воронки» (специфичные условия сверху, дефолт — последним), подготовить шаблон к AJAX-подгрузке (обновление заголовка и хлебных крошек), добавить переключаемые цветовые темы и базовую адаптивность.

## Когда применять

- На одном сайте нужны разные обёртки: например, отдельный шаблон для лендингов, для раздела каталога, для печатной версии или для определённой группы пользователей.
- Кастомный шаблон «не применяется», хотя он привязан, — почти всегда причина в порядке правил (см. ниже про сортировку).
- Нужно дать пользователю/контент-менеджеру выбор цветовой схемы внутри одного шаблона.
- Шаблон используется с AJAX-навигацией (частичная подгрузка страниц), и нужно, чтобы при переходах корректно обновлялись `<title>` и хлебные крошки.

Если нужно создать сам шаблон с нуля — см. [./01-create-template.md](./01-create-template.md). Если интегрируется готовая вёрстка — [./06-integrate-prototype.md](./06-integrate-prototype.md).

## Шаги

1. **Откройте список шаблонов сайта.** «Настройки → Настройки продукта → Сайты → Список сайтов», откройте нужный сайт (стартовый — `s1`), вкладка «Шаблон сайта». Здесь — отсортированный список правил «шаблон + условие + индекс сортировки».

2. **Поймите алгоритм выбора.** Битрикс перебирает правила сверху вниз по индексу сортировки и применяет ПЕРВЫЙ шаблон, чьё условие совпало; дальше список не проверяется. То есть выбор — это «первое совпадение по сортировке», а не «самое точное совпадение».

3. **Расставьте правила по «перевёрнутой воронке».** Самые специфичные условия (по времени, по GET-параметру URL, по точному PHP-выражению) ставьте ВВЕРХ списка; условия по разделам/группам пользователей — в середину; дефолтный шаблон **без условия** (применять всегда) — ОБЯЗАТЕЛЬНО последним. Если широкое/дефолтное правило стоит выше специфичного, оно сработает первым, и специфичный шаблон не применится никогда. Это причина №1 жалобы «мой шаблон не применяется».

4. **Выберите тип условия для каждого правила.** Доступны: путь к папке/файлу (раздел или конкретная страница), PHP-выражение, группы пользователей, период времени, параметр URL. Условие можно оставить пустым — тогда правило применяется всегда (это и есть дефолт, его место — внизу).

5. **Для PHP-условия используйте корректный источник пути.** `$APPLICATION->GetCurDir()` возвращает текущий КАТАЛОГ (директорию), `$APPLICATION->GetCurPage(true)` — текущую СТРАНИЦУ вместе с путём и query-параметрами. Перепутанный вызов даёт условие, которое молча никогда не совпадёт. Готовые шаблоны условий — в сниппете ниже.

6. **Подготовьте шаблон к AJAX-подгрузке (если используется).** Чтобы при частичной загрузке страницы корректно обновлялись заголовок и навигация, в разметке шаблона сайта должны присутствовать элемент с `id="pagetitle"` (туда подставляется заголовок страницы) и элемент с `id="navigation"` (туда — цепочка хлебных крошек). Тип элемента любой (`div`, `span`, `h1`, `p`).

7. **Добавьте переключаемые темы (опционально).** Цветовые схемы оформляют как набор вариантов внутри одного шаблона: параметры темы объявляют в `.parameters.php` шаблона (или хранят выбранную тему опцией модуля), а в `header.php` по выбранному значению подключают нужный CSS темы. Подробнее про чтение настроек шаблона — в разделе «Выбор API».

8. **Заложите адаптивную базу.** В `<head>` шаблона — обязательный `<meta name="viewport" content="width=device-width, initial-scale=1">`; вёрстку ведите по принципу mobile-first (базовые правила для узких экранов, медиазапросы `min-width` для расширения). Без viewport-метатега мобильные браузеры масштабируют страницу как десктопную.

## Рабочий сниппет

Готовые PHP-условия для поля привязки (вкладка «Шаблон сайта»). Каждое — выражение, которое должно вернуть `true`, чтобы шаблон применился.

```php
<?php
// Точный каталог
$APPLICATION->GetCurDir() == "/ru/catalog/phone/";

// По расширению текущей страницы (GetCurPage(true) — путь + query)
substr($APPLICATION->GetCurPage(true), -4) == ".php";
substr($APPLICATION->GetCurPage(true), -5) == ".html";

// По времени (например, чередование оформления по часам)
((date("H") + 6) % 2) == 1;

// OR-комбинация: группа пользователя ИЛИ вхождение раздела
in_array($groupID, $USER->GetUserGroupArray())
    || strpos($APPLICATION->GetCurDir(), "/dir/") !== false;
```

Минимум для AJAX-подгрузки и адаптивности в шаблоне сайта.

Файл: `/local/templates/<id>/header.php`
```php
<?php
if (!defined("B_PROLOG_INCLUDED") || B_PROLOG_INCLUDED !== true) die();
?><!DOCTYPE html>
<html lang="<?= LANGUAGE_ID ?>">
<head>
    <meta charset="<?= LANG_CHARSET ?>">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title><?php $APPLICATION->ShowTitle() ?></title>
    <?php $APPLICATION->ShowHead() ?>
</head>
<body>
<?php $APPLICATION->ShowPanel() ?>

<?php // обновляется при AJAX-подгрузке: хлебные крошки ?>
<div id="navigation">
    <?php $APPLICATION->IncludeComponent("bitrix:breadcrumb", ".default", array(
        "START_FROM" => "0",
        "SITE_ID"    => SITE_ID,
    ), false) ?>
</div>

<main class="workarea">
    <?php // обновляется при AJAX-подгрузке: заголовок страницы ?>
    <h1 id="pagetitle"><?php $APPLICATION->ShowTitle(false) ?></h1>
    <?php // ниже ядро вставит содержимое страницы (work area) ?>
```

Чтение выбранной темы шаблона в `header.php` (если тема хранится опцией).
```php
<?php
use Bitrix\Main\Config\Option;
$theme = Option::get("main", "~bus_theme_".SITE_TEMPLATE_ID, "default");
// допускаем только известные значения (защита от подстановки пути)
$allowed = array("default", "dark", "blue");
if (!in_array($theme, $allowed, true)) {
    $theme = "default";
}
?>
<link rel="stylesheet"
      href="<?= htmlspecialcharsbx(SITE_TEMPLATE_PATH."/themes/".$theme.".css") ?>">
```

## Выбор API

- **Условие привязки** — это реальный PHP, вычисляемый на каждый запрос; источники пути: `$APPLICATION->GetCurDir()` (каталог) и `$APPLICATION->GetCurPage(true)` (страница с query). Это базовый слой, доступный в обеих поддерживаемых версиях API.
- **Хранение выбранной темы:** `\Bitrix\Main\Config\Option::get()/set()` для нового кода; `COption::GetOptionString()` — рабочая обёртка над тем же механизмом, исторически частая в шаблонах.
- **Параметры темы в форме настроек шаблона:** `.parameters.php` шаблона объявляет настраиваемые параметры; значения читаются из `$arTheme`/настроек шаблона в `header.php`.
- **Подключение CSS темы:** через `\Bitrix\Main\Page\Asset::getInstance()->addCss()` (попадёт в объединение/сжатие) либо явным `<link>`, если тема выбирается до вывода `<head>`.
- **Экранирование:** любое значение, попадающее в путь/атрибут, оборачивайте `htmlspecialcharsbx()` и проверяйте по белому списку — имя темы не должно приходить из URL без валидации.

## Проверка

**Режим «только файлы» (без запущенного Битрикса):**
- В PHP-условиях привязки источник пути соответствует смыслу: для каталога — `GetCurDir()`, для страницы/расширения/query — `GetCurPage(true)`.
- В разметке шаблона есть элементы `id="navigation"` и `id="pagetitle"` (если планируется AJAX-навигация).
- В `<head>` присутствует `<meta name="viewport" ...>`.
- Имя темы перед подстановкой в путь проверяется по белому списку и экранируется `htmlspecialcharsbx()`.

**Режим «живой Битрикс»:**
- В списке правил (Сайты → вкладка «Шаблон сайта») дефолтное правило без условия стоит ПОСЛЕДНИМ по сортировке; специфичные — выше.
- На странице, подпадающей под специфичное условие, применяется именно нужный шаблон (видно по разметке/CSS в исходнике страницы).
- На остальных страницах применяется дефолтный шаблон.
- При AJAX-переходе обновляются заголовок (`#pagetitle`) и хлебные крошки (`#navigation`).
- Переключение темы меняет подключённый CSS темы в `<head>`.
- На мобильном (или в эмуляции устройства) страница не масштабируется как десктоп — viewport работает.

## ⚠️ Риски

- ⚠️ **Порядок правил решает всё.** Дефолтное/широкое правило, стоящее по сортировке выше специфичного, перехватывает запрос первым — специфичный шаблон не применится никогда. Дефолт (условие пустое) всегда последний.
- ⚠️ **Путаница `GetCurDir()` / `GetCurPage(true)`** даёт условие, которое молча всегда ложно (или всегда истинно), без ошибки в логах. Сверяйте источник пути со смыслом условия.
- ⚠️ **Имя темы из недоверенного источника** (GET-параметр) без валидации по белому списку и без `htmlspecialcharsbx()` при подстановке в путь к CSS — риск подключения произвольного файла. Всегда фильтруйте.
- ⚠️ **Без `id="navigation"`/`id="pagetitle"`** AJAX-подгрузка оставит старый заголовок и старые хлебные крошки на новой странице.
- Отсутствие viewport-метатега ломает адаптивность: мобильные браузеры рендерят страницу в десктопной ширине и масштабируют.
- ⚠️ Уточнить: предпросмотр шаблона через GET-параметр в обход привязки официально как универсальная фича не задокументирован; для предпросмотра задавайте отдельное правило с условием по этому параметру URL.
