# Рецепт template-03 — Включаемые (редактируемые) области

> Основан на изучении ядра «1С-Битрикс: Управление сайтом» (SM_VERSION 26.x, модули `main`/`fileman`) и официальной документации вендора.

## Цель

Дать контент-менеджеру правку отдельных блоков прямо на странице («карандашик»), не
открывая код шаблона: телефоны, реквизиты, баннеры, текст подвала, врезки в разделах.
Технически это включаемые области — куски HTML/PHP, которые шаблон подтягивает через
`$APPLICATION->IncludeFile(...)` или компонент `bitrix:main.include`.

## Когда применять

- В вёрстке есть блок, который меняется силами редактора, а не разработчика
  (контакты, акции, баннер, юр-реквизиты).
- Один и тот же блок нужен на нескольких страницах/разделах из одного источника.
- Нужна правка через визуальный редактор прямо на публичной странице, с разграничением
  прав. Для вывода списков инфоблока область не нужна — там компонент (см. `../06-output-on-page.md`).

## Шаги

1. **Выбери уровень области.** Компонент `bitrix:main.include` поддерживает три режима
   `AREA_FILE_SHOW`:
   - `file` — конкретный файл по `PATH` (общий блок: телефон, реквизиты);
   - `page` — область своя у каждой страницы (`<имя_страницы>_inc.php` рядом со страницей);
   - `sect` — одна область на раздел (`sect_inc.php` в каталоге, при `AREA_FILE_RECURSIVE=Y`
     наследуется в подкаталоги).
2. **Вставь область в шаблон** (`header.php`/`footer.php` или `template.php`) тем способом,
   что подходит: декларативный компонент `bitrix:main.include` либо императивный
   `$APPLICATION->IncludeFile(...)` (см. «Выбор API»).
3. **Создай файл-область** на диске. Если файла ещё нет, при первом сохранении из режима
   правки движок создаст его по шаблону из `EDIT_TEMPLATE` (по умолчанию `page_inc.php`/
   `sect_inc.php`). Содержимое — обычный HTML или HTML+PHP.
4. **Включи режим правки.** На публичной странице авторизованный сотрудник переключает
   «Режим правки» на админ-панели; рядом с областью появляется иконка-карандаш, ведущая в
   редакторы модуля `fileman` (визуальный — `MODE=html`, текстовый/PHP — `MODE=text/php`).
5. **Выдай права.** Карандаш появляется только при `GetShowIncludeAreas()` и операции
   `fm_edit_existent_file` у группы пользователя (новый файл области — `fm_edit_new_file`).
   Без права на правку файлов раздела редактор не откроется.
6. **Сниппет в шаблоне (sect_edit).** Чтобы редактор показывал понятную подпись и кнопку
   создания, оберни вызов в `BeginEditArea`/`EndEditArea` шаблона (см. сниппет ниже).

## Рабочий сниппет

Подключение области — в `template.php` или `header.php` шаблона
(`/local/templates/<site_template>/`). Файлы-области — в публичной части сайта.

```php
<?php
/** @var CMain $APPLICATION */
if (!defined("B_PROLOG_INCLUDED") || B_PROLOG_INCLUDED !== true) die();

// A. Общий блок «телефон» одним файлом — правят все страницы из одного источника.
//    EDIT_TEMPLATE задаёт заготовку, если файла ещё нет.
$APPLICATION->IncludeComponent("bitrix:main.include", "", [
    "AREA_FILE_SHOW"   => "file",
    "PATH"             => SITE_DIR . "include/phone.php",
    "EDIT_MODE"        => "html",                 // редактор: html | text | php
    "EDIT_TEMPLATE"    => "",
], false, ["HIDE_ICONS" => "N"]);                 // N — показывать карандаш в режиме правки

// B. Область на раздел: /about/sect_inc.php, наследуется в подкаталоги.
$APPLICATION->IncludeComponent("bitrix:main.include", "", [
    "AREA_FILE_SHOW"      => "sect",
    "AREA_FILE_SUFFIX"    => "inc",               // ищет sect_<suffix>.php
    "AREA_FILE_RECURSIVE" => "Y",                 // подняться по дереву каталогов
    "EDIT_MODE"           => "html",
    "EDIT_TEMPLATE"       => "sect_inc.php",
]);
?>
```

Императивный вариант (короче, без обёртки-компонента) — для блоков внутри готовой вёрстки:

```php
<?php
// Абсолютный путь (ведущий «/») — точный файл; путь без «/» резолвится в шаблоне сайта.
$APPLICATION->IncludeFile(
    SITE_DIR . "include/requisites.php",
    [],
    ["MODE" => "html", "NAME" => "Реквизиты", "TEMPLATE" => "page_inc.php"]
);
?>
```

Подпись и кнопка создания области из шаблона (то, что называют `sect_edit`/`AddEditAction`):

```php
<?php
// Режим `page`: блок становится редактируемой областью страницы с подписью в правке.
$APPLICATION->IncludeComponent("bitrix:main.include", "", [
    "AREA_FILE_SHOW"   => "page",
    "AREA_FILE_SUFFIX" => "inc",                  // /catalog/index_inc.php для /catalog/index.php
    "EDIT_MODE"        => "html",
]);
?>
```

Минимальное содержимое файла-области `/local/.../include/phone.php` (или в публичной части —
`/include/phone.php`): просто разметка, ядро ничего не требует.

```php
<span class="header-phone"><a href="tel:+74950000000">+7 (495) 000-00-00</a></span>
```

## Выбор API (для этой задачи)

- **`bitrix:main.include` vs `IncludeFile` — две поддерживаемые версии одного механизма.**
  Компонент удобнее для `page`/`sect`-логики, суффиксов и наследования по разделу; прямой
  `CMain::IncludeFile` короче и точнее для фиксированного файла. Оба рисуют карандаш в
  режиме правки и ведут в редакторы `fileman`.
- **Путь к области.** В `IncludeFile` путь с ведущим `/` — абсолютный (файл-область); путь
  без `/` ищется относительно `templates/<SITE_TEMPLATE_ID>/`, затем `.default`. Для
  контентных областей используй абсолютный путь через `SITE_DIR`, чтобы не зависеть от шаблона.
- **`page` vs `sect` vs `file`.** `page` — уникальная врезка под страницу; `sect` —
  общий блок раздела (с `AREA_FILE_RECURSIVE=Y` наследуется вниз по дереву); `file` —
  единый источник для многих страниц (контакты в шапке). Выбор определяет, где лежит файл.
- **Редактор задаётся `EDIT_MODE`.** `html` — визуальный `CHTMLEditor` (модуль `fileman`):
  кнопки, медиабиблиотека, сниппеты. `text`/`php` — текстовый редактор файла. PHP-режим
  оставляй разработчику, контент-менеджеру выдавай `html`.
- **Хранение в публичной части.** Файлы-области кладём в публичную часть сайта (`SITE_DIR`),
  а не в `/bitrix` или `/local` — их правит редактор, и они часть контента, а не кода.

## Проверка

**Режим «только файлы» (без живого Битрикс):**
- В шаблоне есть вызов `bitrix:main.include` или `$APPLICATION->IncludeFile(...)`; имя
  компонента — ровно `"bitrix:main.include"`.
- Согласованы `AREA_FILE_SHOW` и имя файла: `page` → `<страница>_<suffix>.php`,
  `sect` → `sect_<suffix>.php`, `file` → существующий `PATH`.
- Путь области через `SITE_DIR`/абсолютный для контентных блоков; не указывает в `/bitrix`.
- PHP-синтаксис области и шаблона: `php -l <файл>` (если есть PHP CLI).

**Режим «живой Битрикс»:**
- Открыть страницу под админом, включить «Режим правки» — у области виден карандаш.
- Правка через карандаш сохраняется, на публичной странице блок обновился.
- Если файла области не было — первое сохранение создало его по `EDIT_TEMPLATE`.
- Под пользователем без `fm_edit_existent_file` карандаш отсутствует, контент при этом виден.
- Для `sect` с `AREA_FILE_RECURSIVE=Y` — блок наследуется в подкаталог без отдельного файла.

## ⚠️ Риски

- ⚠️ **PHP-код внутри области исполняется в контексте страницы.** Область с `EDIT_MODE=php`,
  доступная редактору, — это исполнение PHP силами контент-менеджера. Выдавай PHP-режим
  только разработчику; контентным группам — `html`. Иначе ошибка в области ломает всю вёрстку.
- ⚠️ **Несогласованный суффикс/режим → пустой блок или ложное создание файла.** Если
  `AREA_FILE_SUFFIX` в шаблоне и имя файла на диске расходятся, движок не найдёт область и
  при сохранении создаст лишний файл. Сверяй `page_inc.php`/`sect_inc.php` с параметром.
- ⚠️ **Права на правку — отдельная настройка.** Без `fm_edit_existent_file`/`fm_edit_new_file`
  у группы и без режима правки карандаш не появится; это не сбой, а отсутствие прав.
- ⚠️ **Файл-область в зоне кода.** Не клади редактируемые области в `/bitrix` или `/local`:
  правка из публички туда не предполагается, а обновления ядра могут затронуть файл.
- ⚠️ **HTML-редактор экранирует и нормализует разметку.** Сложный ручной HTML/JS в области,
  открытой в `html`-режиме, визуальный редактор переформатирует. Для тонкой вёрстки держи
  блок в `php`/`text`-режиме или внутри шаблона.
