# Интеграция готового прототипа/вёрстки в шаблон Битрикс

## Цель

Превратить готовый статический прототип (экспорт из Figma, готовый HTML+CSS) в рабочий шаблон сайта «1С-Битрикс: Управление сайтом» — сквозным потоком: разрезать разметку на `header.php` + `footer.php` вокруг рабочей области, перенести ассеты в `/local/templates/<id>/`, заменить статичные блоки управляемыми механизмами Битрикса (меню, включаемые области, формы, инфоблоки) и привязать шаблон к сайту.

## Когда применять

- На руках есть законченная вёрстка (HTML/CSS/картинки/шрифты), и нужно «оживить» её в Битриксе.
- Дизайнер отдал готовый прототип, который надо подключить к контенту CMS.
- Нужно перенести один экран (главную) и распространить шапку/подвал на все внутренние страницы.

Если шаблон создаётся с нуля без прототипа — см. `./01-create-template.md`.

## Шаги

1. **Подготовь прототип.** Сведи вёрстку к чистому HTML+CSS: один эталонный экран (обычно главная) с финальной разметкой шапки, контента и подвала. Отметь мысленно границу контента — место, где заканчивается общая «обёртка» и начинается уникальное тело страницы. В терминах визуального создания шаблона эта граница — метка `#WORK_AREA#`: последовательность зон `header → #WORK_AREA# → footer`.

2. **Разрежь по рабочей области — ОДИН логический разрез на месте `#WORK_AREA#`.** Это единственная граница: всё, что выше контента (`<!DOCTYPE>`, `<head>`, открытие `<body>`, шапка, меню, хлебные крошки, открытие контейнера контента), идёт в `header.php`; всё, что ниже (закрытие контейнера, подвал, `</body></html>`), — в `footer.php`. Помни: это «бутерброд» — `header.php` оставляет теги ОТКРЫТЫМИ, `footer.php` их закрывает. По отдельности оба файла невалидны, и это нормально. Точная граница (что отнести к header, что к footer) формально не регламентирована — важно сохранить последовательность `header → рабочая область → footer`.

3. **Вставь обязательные вызовы ядра.** В `header.php`:
   - guard-строка `if(!defined("B_PROLOG_INCLUDED")...)die();`;
   - `<?php $APPLICATION->ShowHead();?>` в начале `<head>` — выводит мета-теги и подключает ВСЕ зарегистрированные CSS/JS; заменяет ручные `<link>`/`<script>`;
   - `<title><?php $APPLICATION->ShowTitle()?></title>`;
   - `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — обязателен для адаптивности;
   - `<?php $APPLICATION->ShowPanel();?>` сразу после `<body>`.
   В `footer.php` — guard, закрытие тегов, подвал, `</body></html>`.

4. **Перенеси ассеты в `/local/templates/<id>/`.** Раскладка:
   - основной CSS оформления шаблона (шапка/подвал/меню/layout) → `template_styles.css`;
   - контентный CSS (стили текста страниц, доступные визредактору) → `styles.css`;
   - картинки → `images/`, шрифты → `fonts/` (или `images/`).
   Оба файла `styles.css` и `template_styles.css` подключаются ядром автоматически — `<link>` на них в `header.php` НЕ добавляй (иначе двойная загрузка). Пути к остальной статике шаблона — только через `<?=SITE_TEMPLATE_PATH?>/...`, не абсолютные/относительные (иначе ломаются при смене ID шаблона).

5. **Замени статичную навигацию на `bitrix:menu`.** Удали `<ul>` меню из вёрстки, вставь компонент с типом меню (`top`/`left`/`bottom`). Подробности — `./02-menu.md`.

6. **Повторяемые блоки → включаемые области.** Логотип, телефоны, реквизиты, соцссылки, копирайт — вынеси в `bitrix:main.include` (режим `AREA_FILE_SHOW => "file"`, путь `SITE_DIR."include/..."`), чтобы контент-менеджер правил их из публички без правки кода. См. `./03-include-areas.md`.

7. **Статичные формы → модуль «Веб-формы».** HTML формы обратной связи замени компонентом веб-формы. См. `../10-feedback-form.md`.

8. **Статичные списки/карточки → инфоблок + компонент.** Повторяющиеся карточки (новости, услуги, товары) перенеси в инфоблок и выводи `bitrix:news.list` / `bitrix:catalog`. Базовый вывод данных на странице — `../06-output-on-page.md`.

9. **Вынеси H1 и хлебные крошки ИЗ редактируемой рабочей области В шаблон.** Заголовок и хлебные крошки — часть обёртки, а не контента страницы: их место в `header.php`, выше `#WORK_AREA#`, чтобы контент-менеджер не мог случайно удалить/задвоить их в визуальном редакторе. Заголовок выводи ровно одним `<h1><?php $APPLICATION->ShowTitle(false)?></h1>` (значение задаётся `$APPLICATION->SetTitle()` в теле страницы/компонентах) — на странице должен быть единственный `<h1>`. Хлебные крошки — компонентом `bitrix:breadcrumb`. Оберни область контента классом-обёрткой из вёрстки. Инлайн-стили/скрипты вынеси в `styles.css`/`template_styles.css`. Шаблон сайта правится только в исходном коде; визуальный редактор применяется к контенту страниц и включаемым областям.

10. **Держи разметку до контента одинаковой на всех типах страниц.** Часть шаблона выше `#WORK_AREA#` (шапка, меню, хлебные крошки, открытие контейнера) должна выводиться байт-в-байт одинаково независимо от типа страницы; различия страниц — только в рабочей области. Это стабилизирует кэширование общих фрагментов и даёт предсказуемую структуру для поисковых роботов.

11. **Привяжи шаблон к сайту.** Заведи `description.php` (имя в списке выбора) — ядро увидит шаблон, т.к. `CSiteTemplate::GetList()` сканирует `/local/templates` и `/bitrix/templates`. Затем в админке «Настройки → Настройки продукта → Сайты → Список сайтов → <сайт> → Шаблон» (или по условию привязки) выбери шаблон.

> Нюанс: переносится именно статический HTML/CSS. Если прототип на React/JSX — портируется визуальный слой (разметка + стили), а данные подключаются компонентами Битрикса; интерактивные JS-блоки перепроверяются вручную.

## Рабочий сниппет/конфиг

`/local/templates/<id>/header.php`:
```php
<?php if (!defined("B_PROLOG_INCLUDED") || B_PROLOG_INCLUDED !== true) die();
use Bitrix\Main\UI\Extension;
Extension::load(["ui.bootstrap4", "ui.fonts.opensans"]); // если нужен штатный Bootstrap 4
?><!DOCTYPE html>
<html lang="<?=LANGUAGE_ID?>">
<head>
    <meta charset="<?=LANG_CHARSET?>">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title><?php $APPLICATION->ShowTitle()?></title>
    <?php $APPLICATION->ShowHead();?>
</head>
<body>
<?php $APPLICATION->ShowPanel();?>

<header class="site-header"><!-- разметка шапки из прототипа -->
    <?php $APPLICATION->IncludeComponent("bitrix:main.include", "", array(
        "AREA_FILE_SHOW" => "file",
        "PATH"           => SITE_DIR."include/company_logo.php",
    ), false);?>

    <?php $APPLICATION->IncludeComponent("bitrix:menu", ".default", array(
        "ROOT_MENU_TYPE" => "top",
        "MENU_CACHE_TYPE" => "A",
    ), false);?>
</header>

<?php $APPLICATION->IncludeComponent("bitrix:breadcrumb", ".default", array(), false);?>
<h1><?php $APPLICATION->ShowTitle(false)?></h1>

<div class="content-wrapper"><!-- обёртка контента, закроется в footer.php -->
```

`/local/templates/<id>/footer.php`:
```php
<?php if (!defined("B_PROLOG_INCLUDED") || B_PROLOG_INCLUDED !== true) die(); ?>
</div><!-- /.content-wrapper -->

<footer class="site-footer"><!-- разметка подвала из прототипа -->
    <?php $APPLICATION->IncludeComponent("bitrix:main.include", "", array(
        "AREA_FILE_SHOW" => "file",
        "PATH"           => SITE_DIR."include/footer_phone.php",
    ), false);?>
</footer>
</body>
</html>
```

`/local/templates/<id>/description.php`:
```php
<?php if (!defined("B_PROLOG_INCLUDED") || B_PROLOG_INCLUDED !== true) die();
$arTemplate = array(
    "NAME"        => "Прототип (имя)",
    "DESCRIPTION" => "Шаблон из интегрированной вёрстки",
    "SORT"        => 100,
    "TYPE"        => "",
);
```

CSS-файлы (создать пустыми, ядро подключит сам):
- `/local/templates/<id>/template_styles.css` — основной CSS шаблона;
- `/local/templates/<id>/styles.css` — контентный CSS.

## Проверка

**Режим «только файлы» (без запущенного Битрикса):**
- структура `/local/templates/<id>/`: есть `header.php`, `footer.php`, `description.php`, `template_styles.css`, `styles.css`;
- в `header.php` присутствуют `ShowHead()`, `ShowTitle()`, `ShowPanel()`, `viewport`; нет ручных `<link>` на `styles.css`/`template_styles.css`;
- ровно один `<h1>` и он через `ShowTitle(false)`;
- пути к статике через `SITE_TEMPLATE_PATH`, нет хардкода абсолютных URL;
- баланс тегов: то, что открыто в `header.php`, закрыто в `footer.php`;
- статичные меню/формы/списки заменены вызовами компонентов, инлайн-стилей в разметке нет.

**Режим «живой Битрикс»:**
- шаблон виден в списке выбора и привязан к сайту; публичная страница рендерится с шапкой и подвалом;
- админ-панель отображается у авторизованного администратора (работает `ShowPanel`);
- CSS подгружается (проверь, что `styles.css`/`template_styles.css` пришли в `<head>` без дублей);
- меню/хлебные крошки строятся динамически, включаемые области редактируются из публички;
- перед сдачей прогнать «Монитор качества» (Настройки → Инструменты → Контроль качества) — обязательные тесты интеграции дизайна должны быть пройдены.

## ⚠️ Риски

- ⚠️ **Работайте в `/local/templates/<id>/`, не в `/bitrix/templates/`.** `/local` имеет приоритет и не затирается обновлениями ядра.
- ⚠️ **Перед привязкой нового шаблона к боевому сайту проверьте его на копии/тестовой странице.** Неполный «бутерброд» (несбалансированные теги, забытый `ShowHead`) ломает вёрстку всех страниц сразу.
- Опечатка в имени шаблона компонента (2-й аргумент `IncludeComponent`) молча подхватит `.default` компонента — внешний вид «уедет» без ошибки.
- Дублирование `<link>` на `styles.css`/`template_styles.css` даёт двойную загрузку CSS.
- Контент, который НЕ должен редактироваться контент-менеджером, оставляйте в коде шаблона по условию (например, только на главной) и не подключайте в него компоненты динамики.
- При портировании React/JSX переносится визуальный слой; интерактивную логику и состояние нужно перепроверить вручную — она не переносится автоматически.
