# Рецепт 05 (deploy): Обновление версий ядра — SiteUpdate, UTF-8, переход между мажорами (24→25→26)

Под-скилл: `1c-bitrix-cms-deploy` • Платформа: «1С-Битрикс: Управление сайтом», ядро 26.x (2026). Модули: Главный (`main`), SiteUpdate, прикладные модули и решения Marketplace.

> **Класс исполнения: E3** — необратимая/привилегированная операция: на проде по умолчанию не исполнять (инструкция человеку или изолированная копия). См. [«0. Право исполнять»](../../conventions.md).

## Цель
Безопасно обновить платформу через штатную систему **SiteUpdate**: соблюсти порядок «ядро → модули → решения», ставить только стабильные обновления, при необходимости перевести сайт на UTF-8 и подготовить окружение (PHP ≥ 8.2, MySQL ≥ 8.0) к мажорному переходу. Результат — актуальное ядро без потери данных и с предсказуемым откатом.

## Когда применять
- Истекает или истёк период обновлений (купон), нужно продлить и накатить накопленные пакеты.
- Переход через несколько мажоров (24 → 25 → 26) или подъём PHP/MySQL до новых минимумов.
- Сайт на однобайтовой кодировке (cp1251): начиная с `main 23.200.0` система обновлений блокирует апдейт до конвертации в UTF-8 (партнёр).
- Плановое регулярное обновление (релизы выходят примерно раз в квартал).

Если задача — только бэкап/восстановление или перенос на другой хостинг, см. `## Связано` и `../../operations.md`.

## Шаги
1. ⚠️ **Полный бэкап перед стартом** (файлы + БД): Настройки → Инструменты → Резервное копирование. `restore.php` со страницы «Список резервных копий» — для восстановления/переноса.
2. **Проверьте окружение.** PHP ≥ 8.2 (с 01.02.2026 пакеты не ставятся на PHP < 8.2; рекомендовано 8.3+, для нового ядра 8.4+). MySQL ≥ 8.0 (ограничение с 01.09.2026; рекомендовано 8.4+). ⚠️ `mbstring.func_overload=0` — при ненулевом значении система обновлений не работает. `default_charset=UTF-8`, `memory_limit ≥ 256M`, OPcache.
3. **Проверьте кодировку.** С `main 24.0.0` поддерживается только UTF-8 (cp1251 не поддерживается). Если сайт на cp1251 — сначала выполните конвертацию (раздел ниже), затем продолжайте.
4. **Активируйте право на обновления.** Право действует 1 год после регистрации лицензии. Если истекло — на вкладке «Активация купона» введите купон продления. Льготный купон применяется заранее или в течение 30 дней после истечения; позже — только стандартное продление.
5. **Проверьте адрес сервера обновлений** в настройках Главного модуля: `www.1c-bitrix.ru` (HTTPS). Опцию «Загружать только стабильные обновления» оставьте включённой (бета — только на тестовом стенде).
6. **Обновляйте по порядку:** сначала сам SiteUpdate, затем ядро (`main`) до промежуточной и далее до актуальной, затем системные модули, затем сторонние модули/решения и шаблоны. Marketplace → «Обновление платформы»; «Установить рекомендуемые обновления» — всё сразу, либо отметьте модули вручную. Сторонние решения — отдельно: Marketplace → «Обновление решений».
7. **Учтите зависимости модулей.** Связанные модули ставятся вместе (например, Торговый каталог ↔ Валюты): выбирайте либо все связанные, либо ни одного. Для точечного выбора целевых версий используйте **Экспертный режим** (колонка «Версия до» — система сама подберёт промежуточные и проверит зависимости, напр. `catalog 25.0.0` требует `ui 25.50.0`).
8. ⚠️ **Переход через мажоры — поэтапно, через промежуточные версии**, не «прыжком». Авторские доработки и сторонние модули проверяйте на совместимость на копии до боевого обновления.
9. **Не прерывайте установку.** Если нужно остановить — кнопка «Остановить установку» завершит текущий модуль, не нарушая целостность.
10. **После обновления — полная очистка кеша, включая композитный:** Настройки → Настройки продукта → Автокеширование → очистить файлы кеша (все опции). Composite-кеш работает поверх обычного и сбрасывается отдельно.
11. **Проверьте ключевые сценарии** (см. `## Проверка`), затем сделайте финальный бэкап и понаблюдайте 3–5 дней (партнёр).

## Рабочий сниппет/конфиг
Минимальные параметры PHP для прохождения обновлений (`php.ini` / пул-конфиг). Подставьте под своё окружение, перезапустите PHP-FPM, перепроверьте «Обновление платформы».

```ini
; --- требования системы обновлений 1С-Битрикс (ядро 26.x) ---
mbstring.func_overload = 0          ; ⚠️ иначе пакеты обновлений не устанавливаются
default_charset       = "UTF-8"     ; обязательно с main 24.0.0
short_open_tag        = On          ; нужно при типовом коде, переходящем на PHP 8
memory_limit          = 256M        ; можно больше
opcache.enable        = 1
; расширения: gd (+freetype), mbstring, xml, zlib, pcre, posix
```

Ключевые элементы перевода сайта на UTF-8 (выполняется **только после бэкапа** ⚠️; предпочтительно — Мастер конвертации `/bitrix/admin/wizard_list.php?lang=ru`, доступен с `23.200.0`). При ручном пути меняются:

```text
.htaccess                         php_value default_charset utf-8
php.ini                           default_charset = "utf-8"
/bitrix/php_interface/dbconn.php  define("BX_UTF", true);   (строки cp1251 убрать)
/bitrix/.settings.php             'utf_mode' => true
БД + все таблицы                  ALTER ... CONVERT TO CHARACTER SET utf8
файлы проекта                     iconv -f cp1251 -t utf8 ...
после                             полный сброс кеша + переавторизация
```

## Проверка
- «Обновление платформы» показывает актуальные версии модулей, новых обязательных пакетов нет.
- Версия PHP ≥ 8.2 и MySQL ≥ 8.0 подтверждены (Настройки → Инструменты → Проверка системы / phpinfo).
- В блоке «Ответ сервера обновлений» дата лицензии актуальна, нет сообщения об истёкшей лицензии.
- После очистки кеша работают: навигация, формы, авторизация, корзина/оплата (для магазина), композитные страницы.
- Журнал ошибок (Настройки → Инструменты → Журнал событий) и серверный error_log без новых фатальных ошибок PHP.
- Сайт в нужной кодировке: страницы и админка отображаются без «крякозябр», `BX_UTF` активен.

## ⚠️ Риски
- ⚠️ **Потеря данных при откате.** Восстановление из бэкапа теряет всё, что накоплено после снятия копии (контент, заказы). Стратегия: либо обновлять на копии и переносить результат на боевой, либо откатываться сразу. Не откатывайте боевой спустя время работы (партнёр).
- ⚠️ **Бан лицензионного ключа при тестировании на копии.** Один ключ на боевом и на копии одновременно даёт несовпадение и блокировку (`[ERROR_WRONG_CODE] Проверка сайта не пройдена`). Снятие — через техподдержку. Используйте отдельный тестовый ключ/окружение, не запускайте систему обновлений с боевым ключом на копии (ВЕНДОР, community).
- ⚠️ **`mbstring.func_overload` ломает обновления.** При ненулевом значении пакеты не устанавливаются — выставьте `0` до старта.
- ⚠️ **Конвертация в UTF-8 — операция повышенного риска.** Только после полного бэкапа и обкатки на отдельной копии. На крупном каталоге Мастер может выдать ошибки структуры БД, требующие ручных SQL-правок из журнала (партнёр). Все сайты на одном ядре должны быть в единой кодировке.
- ⚠️ **Несовместимости при мажорном переходе.** В ядре 2026 удалены базовые классы `CAll*` и JS-расширения `core_fx`/`core_ls` (функционал перенесён в `core`) — прямые обращения к ним приводят к ошибкам (партнёр). Типовые проблемы на PHP 8: конструкторы по имени класса → `__construct()`; статический вызов нестатических методов → `public static`; `count()` на не-массиве; обращение к неопределённым константам; неэкранированные ключи `$arr[key]`.
- ⚠️ **Фиксация старых версий оставляет известные уязвимости.** Экспертный режим позволяет закрепить версию, но вендор рекомендует ставить свежие — иначе накапливаются дыры безопасности и теряется совместимость с интеграциями.
- **Замороженный режим CMS.** После истечения обновлений «Управление сайтом» продолжает работать (публичная часть и магазин функционируют), но без обновлений безопасности и нового функционала. Полного отключения, как у коробочного Битрикс24, для CMS нет (партнёр).
