# Рецепт SEO-07: SEO умного фильтра (catalog.smart.filter)

Под-скилл: `1c-bitrix-cms-seo` • Платформа: «1С-Битрикс: Управление сайтом», ядро 26.x, модуль `iblock`/`catalog`, компонент `bitrix:catalog.smart.filter`.

## Цель
Превратить умный фильтр каталога из «генератора дублей» в источник SEO-трафика: включить **человекопонятные SEF-URL** фасетов, сделать из ценных комбинаций фильтра **индексируемые посадочные страницы** с уникальными `title`/`H1`/`description`, а бесполезные комбинации — закрыть от индекса, не раздувая краулинговый бюджет. Это закрывает большой пласт низко- и среднечастотного спроса («купить красный диван из ткани»), который не покрыть категориями.

## Когда применять
- Интернет-магазин на типовом каталоге Битрикс с умным фильтром; нужно ранжироваться по запросам «бренд + свойство + категория».
- В индекс лезут технические URL фильтра: `/catalog/divany/?arrFilter[...]=...`, `/filter/.../clear/apply/`, тысячи комбинаций с одинаковыми (категорийными) мета-тегами → дубли.
- Нужны посадочные под фильтр с уникальным контентом и мета, а не «голые» отфильтрованные выдачи.

Если задача — только убрать дубли фильтра из индекса (без посадочных), достаточно стратегии индексации из шага 4 + рецепт `06`. Полноценные SEO-страницы фильтра — это шаги 2–3.

## Шаги
1. **Включите SEF умного фильтра.** В комплексном `bitrix:catalog` (режим ЧПУ, `SEF_MODE=Y`) задайте шаблон «URL раздела для умного фильтра»: `#SECTION_CODE#/filter/#SMART_FILTER_PATH#/apply/`. Тогда фасеты дают адреса вида `/catalog/divany/filter/cvet-krasnyj/apply/` вместо GET-портянки. Компонент `bitrix:catalog.smart.filter` подхватывает шаблон через `SEF_MODE=Y` + правило.
2. **Соберите список SEO-комбинаций из семантического ядра.** Не все комбинации нужны в индексе — отберите те, под которые есть спрос (обычно одно-, реже двухсвойственные: «цвет», «бренд», «материал»). Это «быстрые ссылки» / посадочные. Остальное (мультивыбор, редкие пересечения) индексировать не нужно.
3. **Дайте посадочным уникальные мета и контент.** Из коробки страница фильтра наследует мета **категории** → дубли. Варианты:
   - **Готовый модуль** (быстро, без кода): «Сотбит: SEO умного фильтра», «SEO ЧПУ Lite» (dwstroy), «16-в-1 SEO-мастер» (webdebug) — генерируют `title`/`H1`/`description`/`keywords` по шаблонам на основе выбранных значений, ведут свои посадочные и карту.
   - **Кастом**: в `component_epilog.php` шаблона фильтра/каталога считать активные значения фильтра и собрать `SetTitle()`/`SetPageProperty()` по шаблону. Для грамотных русских заголовков (склонение «диван **красного цвета**») подключают `phpMorphy`. Трудозатраты на полноценную реализацию со склонениями и посадочными — порядка **80–100 ч** для среднего магазина (опыт партнёров), больше на каталогах 50k+.
4. **Настройте стратегию индексации (ключевое).**
   - **Открыть для индекса:** кураторские посадочные/«быстрые ссылки» из шага 2 (с уникальными мета).
   - **Закрыть от индекса:** мультивыбор и редкие комбинации, сортировки, пагинацию фильтра, страницу сброса `/filter/clear/apply/`. Способы — `noindex` на странице (предпочтительно, см. рецепт `06`) или `Disallow`/`Clean-param` в robots.
   - **Canonical:** на «шумных» вариациях (сортировка, дозакрытые комбинации) — на каноническую страницу категории или посадочной.
5. **Защитите краулинговый бюджет.** Комбинаторика растёт экспоненциально (свойство с 5 значениями → десятки URL, несколько свойств → тысячи). Без шага 4 робот утонет в дублях. Проверьте логи/Вебмастер: робот не должен большую часть обходов тратить на фильтр.
6. **Учтите конфликт с композитом** (см. рецепт `08`): компонент умного фильтра отключает композитный кэш на своих страницах, а редирект-логика SEO-модулей в событиях `OnProlog`/`OnPageStart` под композитом может не срабатывать. Проверяйте 301 на страницах фильтра при включённом композите.

## Рабочий сниппет

Настройка SEF (через параметры комплексного компонента; в админке — вкладка «Управление адресами страниц»):

```php
// фрагмент параметров bitrix:catalog (SEF включён)
"SEF_MODE" => "Y",
"SEF_URL_TEMPLATES" => [
    "smart_filter" => "#SECTION_CODE#/filter/#SMART_FILTER_PATH#/apply/",
    "section"      => "#SECTION_CODE_PATH#/",
    "element"      => "#SECTION_CODE_PATH#/#ELEMENT_CODE#/",
    // ...
],
```

Кастомные мета посадочной фильтра — в `component_epilog.php` шаблона умного фильтра (есть `$arResult["ITEMS"]` свойств и их выбранные значения):

```php
<?php
if (!defined("B_PROLOG_INCLUDED") || B_PROLOG_INCLUDED !== true) die();
/** @var array $arResult */  /** @var CMain $APPLICATION */

// Собираем выбранные значения активных свойств фильтра.
$selected = [];
foreach ($arResult["ITEMS"] as $prop) {
    foreach ($prop["VALUES"] as $val) {
        if (!empty($val["CHECKED"])) {
            $selected[$prop["NAME"]][] = $val["VALUE"];
        }
    }
}

if (count($selected) === 1) {                       // одно свойство — кандидат на индексацию
    $propName = array_key_first($selected);
    $vals     = implode(", ", $selected[$propName]);
    $section  = $arResult["SECTION"]["NAME"] ?? "Каталог";

    // ⚠️ для грамотного склонения здесь обычно подключают phpMorphy
    $h1    = "{$section}: {$propName} — {$vals}";
    $title = "{$h1} — купить, цена. Доставка по РФ";

    $APPLICATION->SetTitle($h1);                                   // H1
    $APPLICATION->SetPageProperty("title", $title);               // <title>
    $APPLICATION->SetPageProperty("description", "{$section}: {$vals}. Каталог, цены, наличие.");
    // открываем к индексации (мета уникальны) — robots не выставляем
} else {
    // мультивыбор/пустой фильтр — не плодим дубль в индексе
    $APPLICATION->SetPageProperty("robots", "noindex, follow");
    $base = $arResult["SECTION"]["SECTION_PAGE_URL"] ?? ($arResult["FORM_ACTION"] ?? "");
    if ($base) {
        $APPLICATION->SetPageProperty(
            "canonical",
            "https://" . $_SERVER["SERVER_NAME"] . $base
        );
    }
}
```

`robots.txt` — закрыть мусор фильтра под Яндекс (с учётом рецепта `06`):

```text
Clean-param: sort&order /catalog/
Disallow: */filter/clear/apply/
```

## Выбор API
- **SEF фильтра — штатный механизм комплексного компонента** (`SEF_MODE=Y` + `SEF_URL_TEMPLATES["smart_filter"]`). Это и есть «из коробки»: ЧПУ-адреса фасетов Битрикс умеет сам, отдельный модуль для **URL** не нужен.
- **Уникальные мета фильтра «из коробки» НЕ генерируются** — это либо модуль Marketplace, либо код в `component_epilog.php`. Не закладывайте «галочку в настройках» под уникальные `title`/`H1` фильтра.
- **Чтение состояния фильтра** — из `$arResult` компонента `catalog.smart.filter` (`ITEMS` → `VALUES` → `CHECKED`). Для заголовков на уровне категории/элемента работают и наследуемые SEO-свойства (рецепт `03`), но они не знают про активные фасеты — поэтому фильтр обрабатывают отдельно.
- **Закрытие от индекса — `noindex` на странице приоритетнее `Disallow`** для комбинаций, до которых робот уже доходит по ссылкам: `Disallow` оставит «серые» URL в выдаче без сниппета. `Clean-param` (Яндекс) — для сортировок/служебных GET. См. рецепт `06`.
- **`/clear/apply/` — закрывать, не редиректить.** 301 со страницы сброса на категорию ломает кнопку «сбросить фильтр»; правильно — `noindex`/`Disallow`.
- **Готовые модули vs кастом.** Модуль — быстрее и с интерфейсом (посадочные, шаблоны, карта); кастом — гибче и без лицензии, но дороже в разработке (склонения, посадочные, краевые случаи). Выбор — по числу посадочных и бюджету.

## Проверка

**Режим «только файлы» (без живого Битрикса):**
- В параметрах `bitrix:catalog` есть `SEF_MODE=Y` и `SEF_URL_TEMPLATES["smart_filter"]` со `#SMART_FILTER_PATH#`.
- В `component_epilog.php`: чтение `CHECKED`-значений, ветвление «одно свойство → уникальные мета и индекс / мультивыбор → `noindex`+`canonical`». `php -l` без ошибок.
- В `robots.txt` закрыты `*/filter/clear/apply/` и сортировки; посадочные из шага 2 — **не** закрыты.
- Шаблон фильтра лежит в `/local/templates/.../components/bitrix/catalog.smart.filter/`, дистрибутив не правится.

**Режим «живой Битрикс»:**
- Выбор значения фильтра даёт адрес `/catalog/<section>/filter/<...>/apply/` (а не `?arrFilter[...]`), страница отдаёт `200`.
- На посадочной (одно свойство) `<title>`/`<h1>`/`<meta description>` **уникальны** и содержат значение фасета (не равны категорийным).
- На мультивыборе/сбросе в `<head>` есть `noindex` и/или `canonical` на категорию.
- В Яндекс.Вебмастере «Анализ robots.txt»: посадочная разрешена, `/clear/apply/` запрещён. В «Страницах в поиске» со временем растут посадочные, а не технические URL.
- При включённом композите проверьте, что нужные 301-редиректы на страницах фильтра отрабатывают (рецепт `08`).

## ⚠️ Риски
- ⚠️ **Открыть всё подряд = бомба дублей и расход краулингового бюджета.** Комбинаторика фильтра экспоненциальна; в индекс должны попадать только кураторские посадочные. Открытая индексация всех фасетов топит сайт в почти-дублях и размывает релевантность.
- ⚠️ **Мета фильтра не уникальны → каннибализация.** Если все отфильтрованные страницы носят мета категории, они конкурируют друг с другом и с категорией. Уникальные `title`/`H1`/`description` — обязательное условие индексации фильтра.
- ⚠️ **Убрать `/apply/` из URL → 404.** Хвост `/apply/` в SEF-схеме обязателен; «подчищать» его нельзя — страница перестанет открываться.
- ⚠️ **Композит «против» фильтра.** На страницах умного фильтра композит отключается, а редиректы из событий могут не срабатывать под композитом — теряются 301 SEO-модулей. Проверяйте коды ответа на боевом с включённым композитом.
- **Грамматика заголовков.** Машинная склейка «диван цвет красный» читается криво и хуже ранжируется/кликается. Для русского нужны словоформы (`phpMorphy`/словарь) либо ручные шаблоны на ключевые посадочные.
- **Длинные URL.** Мультивыбор даёт очень длинные адреса; их и не индексируют — ещё один довод закрывать мультикомбинации.
- **Пагинация внутри фильтра** — отдельные дубли (`/filter/.../apply/?PAGEN_1=2`); обрабатывайте по рецепту `09`.
