# Рецепт SEO-01: Карта сайта sitemap.xml

Под-скилл: `1c-bitrix-cms-seo` • Платформа: «1С-Битрикс: Управление сайтом», ядро 26.x, модуль `seo`.

## Цель
Сгенерировать карту сайта `sitemap.xml`: файлы из файловой/логической структуры сайта плюс элементы и разделы инфоблоков, с автоматической разбивкой на части и индексным sitemap при большом объёме. Настроить набор карты в `b_seo_sitemap`, запускать пошагово (онлайн) или фоном по агенту/cron, а также авто-добавить строку `Sitemap:` в `robots.txt`. Результат — валидная карта в корне сайта, готовая к регистрации в Яндекс.Вебмастере и Google Search Console.

## Когда применять
- Запускаете корпоративный сайт или интернет-магазин и нужно отдать поисковикам полный список индексируемых URL.
- Каталог/раздел новостей большой (тысячи элементов) — нужна пошаговая генерация без таймаута и автоматическая разбивка ≤ 50 000 URL и ≤ 10 МБ на файл.
- Нужна воспроизводимая настройка набора карты (миграция между стендами, CI), а не только ручная форма в админке.
- Требуется регулярное обновление карты по расписанию (агент/cron) и поддержание строки `Sitemap:` в `robots.txt`.

Если каталог небольшой и обновляется редко — достаточно настроить набор в админке и нажать «Сохранить и запустить»; программный путь ниже нужен для автоматизации и больших объёмов.

## Шаги
1. **Спроектируйте набор карты.** Решите, что входит: статические файлы (маска по умолчанию `*.php,*.html`), какие инфоблоки и что от них (список/разделы/элементы), нужен ли форум. Для корректных URL элементов и разделов **заранее настройте ЧПУ** (см. рецепт 08) — иначе в карту попадут технические адреса либо элементы выпадут.
2. **Создайте запись** в `b_seo_sitemap` через `SitemapTable::add` (или форму «Маркетинг → Поисковая оптимизация → Настройка sitemap.xml»). В `SETTINGS` укажите домен/протокол, имя файла, активные инфоблоки и флаг `ROBOTS` для авто-строки `Sitemap:`.
3. **Запустите генерацию.** Онлайн — пошагово через `\Bitrix\Seo\Sitemap\Generator` (каждый шаг ≤ ~13 сек). Фоном/по расписанию — через `\Bitrix\Seo\Sitemap\Job` + штатный агент.
4. **Проверьте структуру результата.** В корне сайта: индексный `sitemap.xml` (`<sitemapindex>`) со ссылками на части `sitemap_iblock_N.xml`, `sitemap_files.xml`; при превышении лимита части дробятся на `.partN.xml`.
5. **Зарегистрируйте карту** в Яндекс.Вебмастере и Google Search Console; убедитесь, что строка `Sitemap:` присутствует в `robots.txt`.

## Рабочий сниппет
Разовый установочный скрипт: создаёт набор карты и прогоняет генерацию пошагово. Положите в `/local/`, запустите один раз, затем удалите.

Файл: `/local/install/seo_01_sitemap.php`

```php
<?php
// Разовый скрипт. ЗАПУСТИТЬ ОДИН РАЗ, затем УДАЛИТЬ.
require($_SERVER['DOCUMENT_ROOT'] . '/bitrix/modules/main/include/prolog_before.php');

use Bitrix\Main\Loader;
use Bitrix\Seo\Sitemap\Internals\SitemapTable;
use Bitrix\Seo\Sitemap\Generator;
use Bitrix\Seo\Sitemap\Type\Step;

if (!Loader::includeModule('seo') || !Loader::includeModule('iblock')) {
    die('Modules seo/iblock are not installed');
}

$siteId   = 's1';            // ID сайта
$iblockId = 5;               // инфоблок каталога/новостей

// 1. Набор карты в b_seo_sitemap. SETTINGS — конфигурация генератора.
$add = SitemapTable::add([
    'ACTIVE'    => 'Y',
    'NAME'      => 'Основная карта сайта',
    'SITE_ID'   => $siteId,
    'SETTINGS'  => [
        'PROTO'         => 1,                 // 1 = https
        'DOMAIN'        => 'site.ru',         // главное зеркало (без протокола)
        'logical'       => 'N',              // обход файлов по физической структуре
        'FILE'          => 'sitemap.xml',    // индексный файл (_ автоматически → -)
        'DIR'           => ['/' => 'Y'],     // включить корень файлового дерева
        // инфоблок: список + разделы + элементы
        'IBLOCK_ACTIVE'  => [$iblockId => 'Y'],
        'IBLOCK_LIST'    => [$iblockId => 'Y'],
        'IBLOCK_SECTION' => [$iblockId => 'Y'],
        'IBLOCK_ELEMENT' => [$iblockId => 'Y'],
        'ROBOTS'         => 'Y',              // авто-строка Sitemap: в robots.txt
    ],
]);
if (!$add->isSuccess()) {
    die('SitemapTable::add: ' . implode('; ', $add->getErrorMessages()));
}
$sitemapId = $add->getId();
echo "SITEMAP_ID = {$sitemapId}\n";

// 2. Пошаговая генерация. Каждый run() обрабатывает один шаг.
//    Этапы: init(0) → files(10) → iblock_index(20) → iblock(40)
//           → forum_index(50) → forum(70) → index(100).
$gen = new Generator($sitemapId);            // читает SETTINGS из b_seo_sitemap
$gen->init(Step::getFirstStep(), []);        // стартовый шаг + пустое состояние
do {
    $ok    = $gen->run();                    // обработать текущий шаг
    $step  = $gen->getStep();                // 0/10/20/40/50/70/100
    $state = $gen->getState();               // сериализуемое состояние между шагами
    echo $gen->getStatusMessage() . " [step={$step}]\n";
} while ($ok && $step < Step::getLastStep());

echo $ok ? "Карта сгенерирована.\n" : "Прерывание на шаге {$step}.\n";
```

Фоновый/расписанный запуск (вместо ручного цикла) — поставить карту на перегенерацию, остальное доделает штатный агент:

```php
use Bitrix\Seo\Sitemap\Job;

Job::markToRegenerate($sitemapId);   // создаст job и пометит карту на пересборку
// Дальше отрабатывает агент Job::doJobAgent (шаги выполняет $job->doStep()).
```

## Выбор API
Для генерации карты — **`\Bitrix\Seo\Sitemap\Generator` ([D7])**, других путей в ядре нет: модуль `seo` целиком на D7. Выбор внутри D7 — между двумя режимами запуска:

- **`Generator` напрямую (пошаговый цикл)** — для разовой установки, миграций и CI: видите прогресс (`getStatusMessage()`), контролируете завершение по `getStep() < Step::getLastStep()`. Подходит для CLI, где таймаут не давит.
- **`Job::markToRegenerate()` + агент** — для регулярного обновления на боевом сайте: тяжёлую генерацию большого каталога разбивает на шаги по ~13 сек штатный агент (`doJobAgent`), HTTP-запрос не упирается в таймаут. Статусы задачи: `R`/`P`/`F`/`E`.

Внутри генератора используются проверенные legacy-вызовы (это нормально — «две поддерживаемые версии API»): обход файлового дерева через `\CSeoUtils::getDirStructure()`, построение URL элементов/разделов через `\CIBlock::ReplaceDetailUrl()`, punycode домена через `\CBXPunycode::toASCII()`. Их вызывать вручную не нужно — это внутренняя кухня `Generator`.

Для `robots.txt` строку `Sitemap:` добавляет сам генератор при `SETTINGS['ROBOTS']='Y'`. Если нужно управлять файлом отдельно — `\Bitrix\Seo\RobotsFile` ([D7]):
```php
use Bitrix\Seo\RobotsFile;
$robots = new RobotsFile($siteId);
$robots->addRule([RobotsFile::SITEMAP_RULE, 'https://site.ru/sitemap.xml']);
```

## Проверка
**Режим «только файлы» (без живого Битрикса):**
- Скрипт лежит в `/local/install/`, а не в `bitrix/` (не перетрётся обновлением).
- В коде есть: `Loader::includeModule('seo')`, создание набора через `SitemapTable::add` с `SETTINGS` (`DOMAIN`, `PROTO`, `IBLOCK_ACTIVE`), обработка ошибок (`isSuccess()` / `getErrorMessages()`), и пошаговый цикл с условием `getStep() < Step::getLastStep()`.
- `SITE_ID` совпадает с существующим сайтом (по умолчанию `s1`); `iblockId` существует; имя файла в `FILE` без подчёркиваний (или учтена авто-замена `_` → `-`).
- Флаг `ROBOTS => 'Y'` присутствует, если нужна авто-строка `Sitemap:`.

**Режим «живой Битрикс»:**
- Запустите скрипт — он печатает `SITEMAP_ID = N` и строки прогресса по шагам до `step=100`.
- Откройте `https://site.ru/sitemap.xml` — это индексный файл (`<sitemapindex>`) со ссылками на части; части содержат `<urlset>` с `<url><loc>…</loc><lastmod>…</lastmod></url>`.
- Админка: «Маркетинг → Поисковая оптимизация → Настройка sitemap.xml» — набор виден, дата генерации обновилась.
- Откройте `https://site.ru/robots.txt` — присутствует строка `Sitemap: https://site.ru/sitemap.xml` (при `ROBOTS='Y'`).
- Контроль кодом — карта зарегистрирована и активна:
  ```php
  $row = \Bitrix\Seo\Sitemap\Internals\SitemapTable::getById($sitemapId)->fetch();
  // ACTIVE='Y', SETTINGS содержит выбранные инфоблоки
  ```
- URL в `<loc>` — реальные ЧПУ-адреса (не `?ELEMENT_ID=…`); кириллический домен — в punycode.

## ⚠️ Риски
- ⚠️ **Синхронный прогон большого каталога упрётся в таймаут.** `Generator` рассчитан на шаги по ~13 сек. Для тысяч элементов на боевом сайте используйте `Job::markToRegenerate()` + агент, а не один HTTP-запрос с полным циклом.
- ⚠️ **Без настроенного ЧПУ карта получится некорректной.** Ошибки в шаблонах URL инфоблока обычно не видны на странице, но ломают генерацию sitemap и индексацию. Сначала настройте ЧПУ (рецепт 08), затем генерируйте карту.
- ⚠️ **Скрипт-инсталлятор в публичной зоне — риск безопасности и повторного запуска.** Удалите файл сразу после успешного прогона. Перед `SitemapTable::add` проверяйте, что набор с таким `NAME`/`SITE_ID` ещё не создан, чтобы повторный запуск не наплодил дублей.
- **Деактивация/удаление товара не сразу убирает URL из карты** — нужна перегенерация (`Job::markToRegenerate`). Для точечного обновления одного элемента без полной пересборки есть методы `File\Base::appendIBlockEntry()` / `removeEntry($url)`.
- **Стандартная генерация не поддерживает `priority` и `changefreq`** и не даёт событий для фильтрации элементов перед попаданием в карту. Если это критично — нужна кастомная обработка или решение Marketplace.
- **`_` в имени файла карты автоматически заменяется на `-`** (`prepareFileName`, Google не любит подчёркивания) — учитывайте при ссылках на карту и в `robots.txt`.
- **Макросы мультирегиональности** (вида `#CITY_DIR#`) в путях не разворачиваются автоматически — для мультирегиона карту собирают отдельно под каждый домен/сайт.
