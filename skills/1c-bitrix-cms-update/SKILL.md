# 1c-bitrix-cms-update

Безопасное обновление проекта на Битрикс. Принцип: **сначала анализ конфликтов и бэкап, потом применение, всегда на копии/стейдже до прода.** Гейты оркестратора в силе.

## Среда
- «живой Битрикс» → можно прочитать версии, проверить обновления, применить; «только файлы» → анализ конфликтов офлайн (модификации ядра, /local, версии).

## Полный цикл → рецепты (`../../shared/kb/recipes/update/`)
1. интроспекция версий (ядро + модули + приложения Marketplace) → `update/01-introspect-versions.md`
2. поиск доступных обновлений (SiteUpdate + Marketplace) → `update/02-find-updates.md`
3. анализ конфликтов и рисков (правки ядра, переопределения /local, кастом-модули, PHP/СУБД, лицензия) → `update/03-conflict-analysis.md`
4. применение (бэкап → ядро → модули → решения) → `update/04-apply-update.md`
5. пост-проверка и откат → `update/05-rollback-verify.md`
6. диагностика ошибок обновления / симптом → фикс (требования php.ini, неполный апдейт, коды ошибок) → `update/06-troubleshooting.md`
7. миграция кодировки cp1251 → UTF-8 (БД, файлы, сериализованные данные, поиск) → `update/07-utf8-migration.md`
8. обновление решений Marketplace / партнёрских / собственных модулей → `update/08-marketplace-solutions.md`
9. безопасное тест-обновление на копии («Установка для разработки», ERROR_WRONG_CODE, защита ключа) → `update/09-staging-clone-safety.md`

## Инструменты
- `../../shared/scripts/check-conventions.sh <проект>` — поможет в анализе (модификации ядра, код вне /local).
- Бэкап/restore — под-скилл `1c-bitrix-cms-deploy`.

## ⚠️ Главное
Обновление — потенциально разрушительная операция: **обязателен свежий бэкап**, прогон на копии/стейдже, проверка совместимости и лицензии (активный купон обновлений) до применения на проде.

## Завершение
После обновления: сайт поднимается, `check-conventions` без `[FAIL]`, ключевые сценарии работают; иначе — откат.
