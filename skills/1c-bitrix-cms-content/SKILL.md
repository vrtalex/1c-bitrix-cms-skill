# 1c-bitrix-cms-content

Под-скилл сборки контента сайта. Соблюдай гейты оркестратора: весь код — в `/local`, ядро не трогать, правила безопасности, перед сдачей — `check-conventions`.

## Среда
- «только файлы» → создавай корректные файлы и давай инструкцию запуска (агент/CLI/админка).
- «живой Битрикс» → можешь выполнить через PHP CLI/админку и проверить результат в браузере.

## Задача → рецепт (читай нужный из `../../shared/kb/recipes/`)
- понять существующий проект → `01-introspect-project.md`
- создать инфоблок → `02-create-iblock.md`
- добавить свойства → `03-add-properties.md`
- разделы и элементы → `04-sections-elements.md`
- выбрать элементы → `05-query-elements.md`
- вывести на странице → `06-output-on-page.md`
- кастом шаблона компонента → `07-customize-component-template.md`
- комплексный компонент + ЧПУ → `08-complex-component-sef.md` (SEO-аспекты — в **1c-bitrix-cms-seo**)
- пользовательские поля (доп. поля сущностей) → `09-user-fields.md`
- форма обратной связи / приём заявок → `10-feedback-form.md`
- универсальные списки (FAQ, вакансии, реестры) → `11-universal-lists.md`

## Выбор API
По контексту задачи: для вывода штатными компонентами и базовых операций с инфоблоками рабочий путь — legacy (`CIBlock*`); для нового изолированного кода — D7 (`\Bitrix\Iblock\*`). Каждый рецепт указывает рекомендованный путь.
