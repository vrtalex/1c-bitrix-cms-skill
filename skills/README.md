# 1c-bitrix-cms — набор скиллов для сайтов на 1С-Битрикс

Скиллы для создания и доработки сайтов на «1С-Битрикс: Управление сайтом», корпоративные сайты и интернет-магазины.

> **Актуально для ядра 26.x** (выверено на 26.150.0); на более новых ядрах перепроверяйте версионно-зависимые факты в `api-map`/`conventions`.

## Состав — 11 под-скиллов + оркестратор, 75 рецептов (все реализованы)
- **`1c-bitrix-cms`** — оркестратор: определяет среду, маршрутизирует к под-скиллу, объявляет гейты.
- **`1c-bitrix-cms-setup`** — окружение, структура `/local`, установка, CLI, dev-среда (5 рецептов).
- **`1c-bitrix-cms-settings`** — `.settings`/Option, события, агенты/cron, кэш, SMTP (5).
- **`1c-bitrix-cms-template`** — шаблон сайта, меню, включаемые области, Asset/UI, вёрстка (6).
- **`1c-bitrix-cms-content`** — инфоблоки, компоненты, страницы, формы, списки, UF, медиа, интроспекция (16).
- **`1c-bitrix-cms-seo`** — sitemap, robots, мета-шаблоны, ЧПУ/редиректы, OG/schema, дубли/коды ответа, SEO умного фильтра, скорость/Core Web Vitals, пагинация, hreflang/мультиязычность (10).
- **`1c-bitrix-cms-commerce`** — каталог, цены, заказы, оплата/доставка, обмен с 1С (6).
- **`1c-bitrix-cms-deploy`** — git/CI, миграции, бэкап, мультисайт, обновления (5).
- **`1c-bitrix-cms-security`** — проактивная защита, безопасный код, права, аудит, threat intel/CVE (5).
- **`1c-bitrix-cms-update`** — обновление ядра/модулей, проверка совместимости, откат (9).
- **`1c-bitrix-cms-rest`** — REST/webhook, OAuth-приложения, события, прототип-интеграции (4).
- **`1c-bitrix-cms-quality`** — Монитор качества, тестирование, тюнинг, приёмка (4).

**Качество:**
- Оркестратор и под-скиллы используют чёткую маршрутизацию намерение → под-скилл → рецепт.
- Ключевые исполнимые домены проверены на живом «1С-Битрикс» 26.150.0; остальное опирается на статические проверки и изучение ядра и официальной документации. Перепроверяйте под своё окружение.

## Что в бандле
- `skills/` — точки входа (SKILL.md).
- `shared/kb/` — компактная база знаний (overview, conventions, api-map, glossary, operations, recipes/).
- `shared/scripts/` — `check-conventions.sh` (статическая проверка конвенций).
- `shared/assets/` — скелеты компонента/страницы/шаблона.

## Установка
Ставить **единым бандлом**, сохраняя соседство `skills/` и `shared/` (под-скиллы и kb связаны относительными путями `../../shared/...`).

- **Claude (основная цель):** разместить бандл как плагин/проектные скиллы так, чтобы `skills/` и `shared/` оставались сиблингами; скиллы активируются при старте сессии.
- **Codex:** установка тем же механизмом скиллов (сиблинг-раскладка).

> Если платформа изолирует скилл его папкой — применяется запасной вариант: `kb`/`scripts` внутри оркестратора.
