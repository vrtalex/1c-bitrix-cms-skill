# 1c-bitrix-cms-rest

Интеграции сайта на Битрикс через REST и обработчики. Гейты оркестратора в силе; код — в `/local`.

## Среда
- «живой Битрикс» → можно создать вебхук/метод и проверить; «только файлы» → код обработчиков + инструкция.

## Задача → рецепт (`../../shared/kb/recipes/rest/`)
- входящие вебхуки (приём данных от внешних систем) → `rest/01-incoming-webhooks.md`
- объявить свой REST-метод модуля/проекта → `rest/02-custom-rest-method.md`
- исходящая интеграция (отправка во внешний сервис/CRM по событию) → `rest/03-outbound-integration.md`
- локальные приложения и OAuth на коробке → `rest/04-local-apps-oauth.md`

## База знаний
«задача → класс» — `../../shared/kb/api-map.md`; события (для исходящих интеграций) — под-скилл `1c-bitrix-cms-settings`. ⚠️ часть возможностей зависит от модуля `rest` и лицензии; на коробке без модуля `bitrix24` доступны входящие вебхуки и свои методы.

## Завершение
`../../shared/scripts/check-conventions.sh <каталог_проекта>` (в т.ч. проверка `check_bitrix_sessid` и безопасной обработки ввода).
